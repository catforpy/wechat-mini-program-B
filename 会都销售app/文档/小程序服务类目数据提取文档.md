# 小程序服务类目数据提取文档

> 本文档从微信小程序开放服务类目完整文档中提取所有适用于商户小程序模板选择的类目数据
>
> **提取时间**：2026-02-08
> **数据来源**：微信开放文档 - 非个人主体小程序服务类目
> **用途**：为商户提供小程序模板选择的分类体系

---

## 一、类目体系说明

### 类目层级结构

```
一级类目 (横向标签)
  └─ 二级类目 (左侧边栏)
      └─ 三级模板 (右侧卡片网格)
```

- **一级类目**：顶部横向滚动的标签页，用户点击切换
- **二级类目**：左侧垂直列表，显示当前一级类目下的细分领域
- **三级模板**：右侧卡片网格，显示具体的小程序模板供商户选择

### 模板卡片结构

每个模板包含：
- **name**：模板名称（如：快餐店、火锅店）
- **desc**：简短描述（如：连锁快餐、火锅烧烤）

## 三、数据结构定义

### TypeScript 接口定义

```typescript
// 三级类目模板
export interface TemplateInfo {
  name: string      // 模板名称
  desc: string      // 简短描述
}

// 二级类目
export interface SecondLevelCategory {
  name: string                    // 二级类目名称
  qualification?: string          // 资质要求
  scope?: string                  // 适用范围
  templates: TemplateInfo[]      // 三级类目（小程序模板）
}

// 一级类目
export interface FirstLevelCategory {
  name: string                      // 一级类目名称
  secondLevel: SecondLevelCategory[] // 二级类目列表
}
```

## 五、注意事项

1. **资质要求**：部分类目需要提供相应资质证明，模板展示时应标注资质要求
2. **适用范围**：每个二级类目都有明确的适用范围，需在模板详情中说明
3. **动态更新**：微信小程序服务类目会随政策和平台要求变化，需要定期更新
4. **内测类目**：部分类目处于内测阶段，暂不支持线上申请

## 六、后续开发建议
