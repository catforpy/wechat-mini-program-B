## 六、数据结构改进

### 新增字段

```typescript
export interface SecondLevelCategory {
  name: string                    // 二级类目名称
  qualification?: string          // 资质要求
  scope?: string                  // 适用范围
  restrictions?: string           // 不得涉及的内容（仅境外主体）✨ 新增
  templates: TemplateInfo[]      // 三级类目（小程序模板）
}
```

### 数据特点

1. **个人主体**
   - 资质要求多为"/"（无需特殊资质）
   - 适用范围明确，包含"注"说明不可涉及的内容
   - 部分类目包含三级模板（如丽人服务的10个细分服务）

2. **境外主体**
   - 新增 `restrictions` 字段，记录"不得涉及"的内容
   - 资质要求较为复杂，常需多项证明文件
   - 包含"内测"类目（如旅游退税、演出票务、证券、医疗资讯）
   - 部分类目仅对特定区域开放（如共享单车仅限新加坡、日本）

3. **企业主体**
   - 数据量最大（24个一级类目，约300+个二级类目）
   - 资质要求最复杂，常需多选一或同时提供多项证明
   - 包含大量政务民生类目（44个二级类目）
   - 医疗服务类目详细（18个二级类目，包含公立、私立、互联网医院等）

## 七、使用方式

### TypeScript 导入

```typescript
import {
  getCategoriesByEntityType,
  EntityType,
  companyCategories,
  personalCategories,
  overseasCategories
} from '@/data/categories-by-entity'

// 根据主体类型获取类目
const categories = getCategoriesByEntityType(EntityType.PERSONAL)

// 直接使用
const allPersonalCategories = personalCategories
```

### 数据访问示例

```typescript
// 获取个人主体的"丽人服务"类目
const beautyService = personalCategories
  .find(cat => cat.name === '生活服务')
  ?.secondLevel.find(sub => sub.name === '丽人服务')

// 访问三级模板
beautyService?.templates.forEach(template => {
  console.log(`${template.name}: ${template.desc}`)
})
// 输出：
// 美甲: 美甲服务
// 美容: 美容服务
// 美睫: 美睫服务
// ...（共10个）
```

### 境外主体特有字段

```typescript
// 获取境外主体的限制条件
const logisticsService = overseasCategories
  .find(cat => cat.name === '快递业与邮政')
  ?.secondLevel.find(sub => sub.name === '快递、物流')

console.log(logisticsService?.restrictions)
// 输出：
// 不得涉及：
// 1、空中交通管制公司；
// 2、邮政公司。
```
